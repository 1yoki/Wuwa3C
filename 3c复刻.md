# UE5 四周复刻《鸣潮》3C执行计划

## 1. 项目目标

在四周内完成一个具有《鸣潮》风格的第三人称3C纵向切片，重点展示：

* 流畅的地面移动与空中移动
* 冲刺、跳跃、二段跳、空中钩锁
* 跑墙、攀爬、抓边、翻越
* 普通攻击、空中攻击、闪避、完美闪避
* 敌人攻击、弹刀、受击、击飞
* 软锁定、硬锁定与目标切换
* 动作输入缓存、取消窗口、打断优先级
* 多种镜头模式及平滑切换
* 两个角色共用同一套移动与探索框架
* 一个可进行弹刀交互的精英敌人或Boss

最终目标不是完整复制《鸣潮》，而是实现一套：

> 可扩展、可配置、可演示、能够体现客户端开发能力的第三人称动作游戏3C框架。

# 4. 开发原则

## 4.1 每天必须保持项目可运行

禁止连续几天只搭架构、不产出实际效果。

每天结束前必须至少做到：

* 项目可以编译
* 角色可以正常生成
* 基础移动没有被破坏
* 新功能可以在测试关卡触发
* Git提交一次可运行版本

## 4.2 动画不决定Gameplay逻辑

动画蓝图只负责表现。

Gameplay状态由以下系统决定：

```text
CharacterMovementComponent
ActionRouter
CombatComponent
TraversalComponent
Gameplay Tags
```

动画通过变量读取Gameplay状态，而不是反向控制角色核心逻辑。

## 4.3 输入不直接执行动作

错误流程：

```text
攻击键
→ 直接播放Montage
```

正确流程：

```text
输入
→ 生成输入意图
→ Action Router检查
→ 判断能否执行
→ 执行、缓存或拒绝
```

## 4.4 位移权威必须唯一

每个动作必须明确由谁控制Capsule位移：

* CharacterMovement
* Root Motion
* Motion Warping
* Root Motion Source

禁止同一个动作同时使用：

```text
LaunchCharacter
SetActorLocation
Montage Root Motion
CharacterMovement Velocity
```

否则容易产生位移冲突。

## 4.5 数据驱动优先

以下参数不要硬编码：

* 移动速度
* 加速度
* 跳跃高度
* 闪避距离
* 钩锁速度
* 跑墙速度
* 攻击吸附距离
* 输入缓存时长
* 取消窗口
* 镜头FOV
* 镜头距离

统一放入：

```text
Data Asset
Data Table
Curve
Character Definition
```

### 任务

实现两种钩锁。

### 自由钩锁

# 12. 风险控制

## 12.1 动画资源不足

解决方式：

* 使用UE Mannequin动画
* 使用免费动画资源
* 使用IK Retargeter重定向
* 先使用占位动画完成逻辑
* 后期替换，不等待最终动画

## 12.2 跑墙和攀爬穿模

检查：

* Capsule是否由MovementComponent移动
* 是否错误使用SetActorLocation
* 墙面法线是否稳定
* 是否每帧重新约束离墙距离
* 是否进行了碰撞Sweep

## 12.3 状态无法退出

每个状态统一实现：

```text
Enter
Update
CanExit
Exit
ForceExit
```

退出时恢复：

* MovementMode
* Gravity Scale
* Collision
* Rotation Mode
* Input Block Tags
* Camera Mode
* Root Motion
* Montage
* Gameplay Tags

## 12.4 镜头抖动

检查：

* Camera Lag和代码插值是否重复
* 锁定目标点是否使用Root Bone
* 是否每帧在多个模式之间切换
* 镜头碰撞是否快速伸缩
* 角色和目标权重是否平滑变化

## 12.5 攻击吸附过强

限制：

```text
最大距离
最大角度
最大位移
最大转向速度
可见性检测
障碍物检测
攻击阶段限制
```
