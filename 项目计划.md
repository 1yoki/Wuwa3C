### 1.2 每日固定节奏（4～6 小时）

```text
20 分钟  拉取/打开工程、回归昨天的最小动作链
30 分钟  明确当天数据结构、状态拥有者和退出路径
```

## Day 5：输入缓存、动作路由与地面冲刺

### 当日目标

建立后续探索和战斗共用的最小动作骨架，并用 Sprint Pressed 的前冲/后撤完成第一个端到端动作。只抽象已需要的能力，不做万能状态机。

### C++ 类型与组件

- `FWuwaInputCommand`：InputTag、PressedAt、ValidDuration、Direction、Sequence。
- `UWuwaInputBufferComponent`：Push、Peek、Consume、Expire、ClearByTag。
- `FWuwaActionRequest/Result/EndReason`。
- `UWuwaActionDefinition`：ActionTag、Priority、Required/Blocked/Granted Tags、BufferTime、Cancel Rules、MovementPolicy、Montage。
- `UWuwaActionRouterComponent`：Request、CanStart、CancelCurrent、FinishCurrent、TryConsumeBuffer。
- 轻量 `UWuwaStateTagComponent`，或由 Router 内封装来源计数的 Tag 容器。

### 迁移步骤

1. Controller 把 Sprint/Dodge/Attack/Grapple 等单帧布尔意图转换为 Command。
2. Move/Look 继续走持续快照；Sprint Pressed 进入 FIFO，长按不重复生成命令。
3. Character 把 Command 交给 Router；未实现的 Attack/Grapple 返回可调试的 `NoExecutor`，不能直接播放动画。
4. 创建 `DA_Action_DashForward_Ground` 与 `DA_Action_Backstep_Ground`。
5. Sprint Executor 在接地且有输入时前冲，无输入时沿角色后方后撤；按下时快照方向。
6. 前冲正常完成且仍有移动输入时调用 `EnterSprintRun`；后撤不进入高速跑步。
7. 动作结束、Montage 中断、受击强断和 Owner EndPlay 都调用统一 `FinishCurrent`。

### 地面冲刺规则

- 有输入：沿按下瞬间的相机空间移动方向前冲，完成后进入高速跑步。
- 无输入：沿角色后方短距离后撤，不进入高速跑步。
- 只允许接地开始；冷却、动作冲突和无效方向给出拒绝原因。
- 当天选择 **Root Motion Montage** 或 **代码/Root Motion Source 位移** 之一并记录到 Definition，禁止混用。
- 点按和长按只触发一次；按键释放不取消动作，也不退出前冲后的高速跑步。
- 添加 `State.Action.Dash`、`State.Locomotion.Sprinting`、`Block.Input.Move`；无敌窗口留给 Day 20。

### Debug 与测试

- HUD/日志显示当前动作、输入命令剩余时间、Request Result、End Reason。
- 在冲刺末尾再次按 Sprint：按规则缓存或拒绝，不重复叠放 Montage。
- 冲刺中从平台跌落、停止 Montage、销毁角色：输入、旋转、相机和标签均恢复。
- 缓存过期后不执行；同一 Sequence 不能消费两次。
- 前冲结束后保持移动进入 Sprint Speed；输入归零、离地或阻止标签出现时退出。
- 后撤完成后地面移动与跳跃回归正常。

### 交付与提交

- 交付：可复用 Input Buffer、Action Router、数据驱动前冲/后撤与高速跑步闭环。
- 建议提交：`feat: add action router and contextual sprint dash`

## 5. 最终架构验收清单

### 解耦合

- [ ] PlayerController 只生成输入意图/命令，不直接播放 Gameplay Montage。
- [ ] Character 主要负责组件装配与 Definition 初始化。
- [ ] Action Router 不实现移动物理、攻击 Trace 或镜头插值。
- [ ] Traversal 的 Query、Evaluate、Execute 已分离。
- [ ] Targeting 不直接控制镜头或攻击。
- [ ] Animation/UI/VFX/Camera 通过只读状态或事件消费 Gameplay 事实。
- [ ] 第二角色没有复制 Gameplay C++ 类。

### 数据驱动

- [ ] Movement、Traversal、Targeting、Camera 参数来自 Profile。
- [ ] 动作准入、优先级、取消和位移策略来自 Action Definition。
- [ ] 攻击数值、Trace、吸附和反馈键来自 Attack Definition/Set。
- [ ] 角色差异由 Character Definition 组合。
- [ ] 关键 Data Asset 有校验与安全默认值。
- [ ] Runtime State 不回写 Data Asset。

### 单一权威与恢复

- [ ] Movement Component 是 Capsule 位置、速度和 MovementMode 权威。
- [ ] Action Router 是当前独占动作权威。
- [ ] Attribute Component 是 Health/Posture/Stamina 权威。
- [ ] Targeting Component 是当前目标权威。
- [ ] Camera Manager 是最终 POV 权威。
- [ ] 每个动作在 Completed/Cancelled/Interrupted/Failed/Destroyed 下都能释放资源句柄。
- [ ] 不存在 Root Motion、SetActorLocation、LaunchCharacter 和 Movement Velocity 的同动作冲突。
